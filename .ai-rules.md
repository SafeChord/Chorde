# Chorde (Infrastructure) AI Rules

This directory contains the platform and operations code (K3s, ArgoCD, GitOps) for SafeChord.

## 1. Directory Structure
Understanding the internal layout of `Chorde/` is crucial for infrastructure management.

```text
Chorde/
├── cluster/      # Cluster provisioning and configuration (e.g., k3s, Ansible).
├── gitops/       # GitOps root, manifests, and stage definitions (ArgoCD).
├── legacy/       # Deprecated or old cluster configurations.
└── scripts/      # Automation scripts (e.g., ops/seal.sh, infra testing).
```

## 2. Core Policies
- **SafeZone-Free Zone**: Do NOT place any application-specific Helm charts or deployment configurations here. This repository is strictly for underlying infrastructure.
- **GitOps Only**: Permanent modifications to the cluster MUST be done via manifests and committed to the repository.

## 3. Secret Management
- **Never commit plain-text secrets.**
- **Workflow**:
  1. Create unsealed manifests ONLY in explicitly git-ignored directories.
  2. Run `scripts/ops/seal.sh <path>` to generate the `sealed-secret.yaml` file.
  3. Commit only the sealed secrets.
